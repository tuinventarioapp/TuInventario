# Patrones de modelado

## Item

Modelar `item` con flags o una estrategia clara para diferenciar consumible y prestable sin duplicar entidades innecesariamente.

## Movimiento

Modelar movimientos con tipo, cantidad, origen, destino, motivo y responsable.

## Prestamo

Separar solicitud de prestamo, prestamo efectivo y detalle por item.

## Auditoria

Guardar actor, entidad, accion, contexto y timestamp.
