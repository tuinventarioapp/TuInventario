## Actualizar

- docs funcionales y tecnicos impactados

## Entregar

- resumen corto
- lista de archivos actualizados
- pruebas ejecutadas
- gaps conocidos que siguen pendientes

## Evitar

- documentar features hipoteticas
- dejar contradicciones entre codigo y docs
- olvidar sincronizar documentacion espejo
