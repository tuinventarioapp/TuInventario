# Definition of Done

Una funcionalidad se considera terminada solo si cumple todos los puntos siguientes.

## Producto

- resuelve un problema real del flujo objetivo;
- respeta reglas de negocio;
- no introduce ambiguedad operativa;
- tiene mensajes de error comprensibles.

## Codigo

- sigue la arquitectura definida;
- no mezcla logica de negocio con presentacion;
- maneja errores esperables;
- mantiene nombres claros y consistentes.

## Datos

- valida organizacion, permisos y estado;
- no rompe historial;
- no deja stock inconsistente;
- incluye migracion o ajuste de datos si aplica.

## Calidad

- cuenta con pruebas relevantes;
- pasa lint y formateo;
- actualiza documentacion afectada;
- deja instrucciones de validacion manual.

## Evidencia minima

- resumen de cambios;
- archivos modificados;
- pruebas ejecutadas;
- riesgos pendientes;
- siguiente paso recomendado.
